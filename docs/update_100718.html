*This is how I fixed it.

  1. Find the SHA of the last commit that had a "good" copy of the file that you want to recover. You only need the seven-digit number, not the full number, which is good, because copying from anywhere but the terminal and pasting into the terminal is horrible on this thing. There are terminal commands and shortcuts for this that I might edit in later, but in my case I just browsed to the commit history page on my repository and found the right number.
  2. Use this command:
    $git checkout [SHA] dir/file
    [SHA] is the seven (or forty) digit number; dir/file is the path to whatever file you're grabbing. That will basically download the correct file to your directory.
  3. Use git add, git commit and git push as you normally would, to create a new commit that will overwrite the missing or incorrect version in the repository.

It took me a while to find a straight answer about this problem. I hope that writing it down here will help someone.
